# Why most PM frameworks collapse inside agentic systems

Most PM frameworks aren't built for agentic systems — they're built for human-only workflows. RICE, Kano, OKRs, even classic discovery: each one was sharpened against a world where humans were the only artifact producers.

That world is gone.

The shorthand I use is "framework collapse." A framework is a set of choices about which questions matter. When the underlying medium changes — from human workflows to agentic ones — the questions change. The frameworks either adapt or they collapse.

So I built Lattice. Not a product but a protocol — a deliberate, facilitator-mediated process that recontextualizes how a PM team works through a build using AI as the primary mediation surface.

The headline claim: Lattice changes what the PM measures. Instead of shipping velocity, you measure question quality. Instead of standups, you measure delegation density. Instead of retros, you measure pattern-capture rate.

Three teams ran it. All three reported their agentic-product cycle time dropped by half.

Pricing is $7,500 for a standard engagement, $12,500 for an extended one. Facilitator-mediated by design.

The counter-argument is fair: a self-serve product would reach more PMs faster. True. But the protocol exists because most teams skip the deliberate part — the part where you decide what's worth asking before you decide what's worth building. Without a facilitator, the protocol collapses into another notebook of unread frameworks.

If you're a PM building inside an agentic product, here's what I'd ask you to do this week: take one weekly artifact you produce — a PRD, a spec, a competitive read — and run it through three rounds where an agent has to defend every load-bearing claim. The agent will fail. That failure is the diagnostic.

Read the protocol entries. Book the engagement track.
