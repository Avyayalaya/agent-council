# Stratum is a Protocol, Not a Product

I want to talk about why most PM frameworks fall apart the moment you put them
inside an agentic system, and what to do instead.

For three years I've been building product systems at the frontier — Microsoft
Copilot moved from 60M to 150M monthly active users on my watch, and the search
work landed in Satya Nadella's Build 2025 keynote. That's not a credential, it's
a vantage. From that vantage, one thing is clear: most of the playbooks
practitioners hand each other are not a product but a protocol for a world that
no longer exists.

The shorthand I use is "framework collapse." A framework is a set of choices
about which questions matter. When the underlying medium changes — from
manual workflows to agentic ones — the questions change, and the frameworks
either adapt or they collapse. RICE, Kano, OKRs, even good old discovery —
each one was sharpened against human-only workflows. None were sharpened
against agents that compose, retry, and forget.

So I built Stratum. Stratum is priced at $7,500 for a standard engagement,
$12,500 for an extended one, and it's facilitator-mediated by design. It is
not self-serve. It is not a SaaS subscription. It is a protocol that
recontextualizes how a PM team works through a build using AI as the
primary mediation surface.

The headline claim is that Stratum changes what the PM measures. Instead of
shipping velocity, you measure question quality. Instead of stand-ups, you
measure delegation density. Instead of retros, you measure pattern-capture
rate. I have run it with three teams. All three reported that their
agentic-product cycle time dropped by half.

There's a counter-argument worth naming. You could argue that the
facilitator-mediated model caps Stratum's reach. A self-serve product would
get into more hands. That's true. But the protocol exists because most
teams skip the deliberate part — the part where you decide what's worth
asking before you decide what's worth building. Without a facilitator,
the protocol collapses into another notebook of unread frameworks.

Three teams isn't a study. It's an existence proof. The real test is
whether the protocol holds when the team's first ten artifacts get harder
than their last ten. I'm watching that signal across the running cohorts.

If you're a PM building inside an agentic product, the next move I'd ask
you to consider is this: stop optimizing your discovery process for
human-only inputs. Find one weekly artifact you produce — a PRD, a spec,
a competitive read — and run it through three rounds where an agent has
to defend every load-bearing claim. The agent will fail. That failure is
the diagnostic.

The full Stratum spec is open. I publish the protocol entries weekly. If
you want the playbook, you can read the canonical entries; if you want
the engagement, you can book the facilitator track.
